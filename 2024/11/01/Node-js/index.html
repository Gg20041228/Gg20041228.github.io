Node.js

  • 运行在服务器端的js
  • 用来编写服务器
  • 特点：
    • 单线程、异步、非阻塞
    • 统一API

Node.js 和 JavaScript 的区别:

JavaScript: ECMAScript DOM BOM

Node.js: ECMAScript DOM BOM

进程和线程

进程（厂房）：程序运行的环境

线程（工人）：线程是实际进行运算的东西

同步

  • 通常情况下代码都是自上而下一行一行执行的
  • 前边代码不执行后边代码也不执行
  • 同步代码执行会出现阻塞的情况
  • 一行代码执行慢会影响整个程序的执行

那是如何解决同步问题的呢？不同的编程语言有不同的方法

Java python :通过 多线程 来解决

Node.js :通过 异步 方式来解决

  • 不会阻塞其它代码的执行

  • 代码的可读性差
  • 可调试性差

那如何去解决这个问题？归根结底是因为异步函数需要通过回调函数来得到我们的结果，这时我们便需要有一个东西来代替回调函数让我们得到结果，因此我们引入了 Promise
